Herramientas y técnicas para incrementar la calidad en el código de tu proyecto

En esta conferencia nos nos dieron muchos tips para aplicarlos en nuestros proyectos.

  • Mantener a los desarrolladores felices.
  • Mantenerlos relajados.
  • No existe el software perfecto, pero sí el de gran calidad.
  • La calidad de un software se puede medir de muchas maneras.
    • Debe cumplir con su proposito.
    • Informa adecuadamente cuando ocurre un problema.
    • Tiene pruebas.
    • Se entiende su intenció con sólo leer el código.
    • Sigue un estilo de código desde el inicio.
    • Utiliza un flujo de colaboración adecuado.
    • Implementa correctamente los patrones de diseño.
    • Se puede utilizar intuitivamente.
  • Realiza muchas pruebas.
    • Unitarias.
    • Integración.
    • GUI.
  • Que el código tenga buena legibilidad y estilo.
  • Source control
    1. Definir rama estable.
    2. Definir rama para ambiente de integración.
    3. definir el estándar para nombrar las ramas.
    4. Definir un estándar para los commits.
    5. Definir una persona para la revisión de código.
